Sistema de Inventario por WhatsApp

Propuesta técnica y plan de desarrollo
Fecha de emisión: 14 de abril de 2026  •  Validez de la propuesta: 30 días

1. Resumen ejecutivo

Se propone desarrollar un sistema que permita gestionar el inventario de la tienda directamente desde WhatsApp, mediante mensajes de texto y notas de voz. El personal no requerirá instalar aplicaciones adicionales ni aprender a usar software nuevo: basta con enviar un mensaje en lenguaje natural (por ejemplo, "agrega 20 cocas de 600") y el sistema actualiza el inventario automáticamente, respondiendo con una confirmación.

La arquitectura se basa en herramientas maduras y de bajo costo operativo, lo que permite un tiempo de entrega corto y mantenimiento simple a largo plazo.

2. Tecnologías propuestas

2.1 Canal de entrada

TecnologíaRol en el proyecto
WhatsApp Cloud API (Meta)Canal oficial de mensajería. Recibe y envía mensajes de texto y notas de voz.
Whisper (Groq)Transcripción de notas de voz a texto con alta precisión en español.

2.2 Orquestación e inteligencia

TecnologíaRol en el proyecto
n8n (autoalojado)Motor de flujos que conecta WhatsApp, la IA y la base de datos. Visual y mantenible.
Claude Haiku 4.5 (Anthropic)Interpreta el mensaje del usuario, identifica la intención (consultar, agregar, reportar) y extrae el producto y la cantidad.
Embeddings (OpenAI)Búsqueda difusa de productos para tolerar errores de transcripción o escritura.

2.3 Datos y almacenamiento

TecnologíaRol en el proyecto
PostgreSQL (autoalojado)Base de datos del inventario, desplegada en la misma instancia EC2 mediante Docker. Robusta, estándar de la industria y sin costos de licencia.
pgAdmin / AdminerPanel web para consultar y editar productos manualmente cuando sea necesario.
Tabla de auditoríaRegistro automático de cada movimiento: quién, qué, cuándo y cuánto.

2.4 Infraestructura

TecnologíaRol en el proyecto
AWS EC2 (t3.medium)Servidor virtual que aloja n8n y PostgreSQL en contenedores Docker. 2 vCPU y 4 GB de RAM, suficiente para el volumen previsto con margen de crecimiento.
AWS EBS (gp3, 30 GB)Almacenamiento persistente para la base de datos, respaldos y volúmenes de Docker.
Docker + Docker ComposeOrquestación de los servicios (n8n, PostgreSQL, reverse proxy) para despliegue reproducible y actualizaciones limpias.
Caddy / Nginx + Let's EncryptReverse proxy con certificado SSL gratuito y renovación automática. Endpoint seguro para los webhooks de WhatsApp.
Snapshots EBS automáticosRespaldos diarios de la base de datos y configuración, con retención de 7 días.

4. Tiempo estimado de desarrollo

El proyecto se entregará por fases incrementales. Cada fase deja una versión funcional que el cliente puede usar y validar antes de avanzar a la siguiente.

Fase 1 — Configuración e integración base (7 h)
Alta de WhatsApp Business, provisión del EC2 t3.medium, despliegue de n8n y PostgreSQL mediante Docker Compose, configuración de reverse proxy con SSL, creación del esquema de base de datos y webhook operativo.
Fase 2 — Flujo de texto (10 h)
Interpretación de mensajes de texto, consultas de stock, alta de inventario, respuestas automáticas.
Fase 3 — Notas de voz (5 h)
Transcripción de audios con Whisper y búsqueda difusa de productos para tolerar errores de transcripción.
Fase 4 — Reportes, confirmaciones y auditoría (5 h)
Reportes bajo demanda, alertas de stock bajo, confirmación de acciones sensibles, log de auditoría.
Fase 5 — Pruebas, capacitación y entrega (3 h)
Pruebas con usuarios reales, ajustes finales, capacitación al personal, documentación de uso.

Esfuerzo total estimado: 30 horas de desarrollo, distribuidas de forma incremental a partir de la firma del acuerdo y la entrega de los accesos requeridos (cuenta de AWS, cuenta de Meta Business, cuenta de Google). El calendario concreto (horas por semana) se acordará con el cliente en función de la urgencia y la disponibilidad para validaciones intermedias.

5. Entregables

  • Sistema funcional desplegado y operativo en producción sobre AWS EC2.
  • Base de datos PostgreSQL con el inventario inicial cargado y esquema documentado.
  • Configuración de Docker Compose versionada y lista para reproducir el entorno.
  • Política de respaldos automáticos (snapshots EBS) configurada.
  • Manual de uso en español dirigido al personal de la tienda.
  • Documentación técnica para futuras ampliaciones.
  • Capacitación remota de 1 hora al personal que usará el sistema.
  • 30 días de soporte post-entrega para ajustes y correcciones.

6. Supuestos y consideraciones

  • El cliente facilitará una línea telefónica dedicada para WhatsApp Business.
  • El cliente proveerá el listado inicial de productos en formato digital.
  • El cliente proveerá la cuenta de AWS donde se desplegará la infraestructura, o autorizará la creación de una nueva a su nombre.
  • Los costos mensuales son facturados directamente por los proveedores de servicios al cliente (AWS, Meta, Anthropic, Groq, Google); el desarrollador no intermedia pagos recurrentes.
  • El mantenimiento del sistema operativo de la instancia EC2 (parches de seguridad, actualizaciones de Docker) puede incluirse en un contrato de soporte mensual opcional tras los 30 días de soporte inicial.
  • Ampliaciones posteriores (multi-sucursal, integración con punto de venta, app móvil, alta disponibilidad) se cotizan por separado.
